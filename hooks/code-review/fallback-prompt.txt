Review the file referenced in <context> for real defects, restricting

Look for, in priority order:
1. Bugs and correctness defects: null/undefined access, off-by-one,
   inverted conditions, wrong case discrimination, unreachable code.
2. Security: SQL/command/path injection, XSS, hardcoded secrets,
   unsafe deserialization, weak randomness, missing authn/authz checks.
3. Error handling: swallowed exceptions, missing validation, empty
   catch blocks, errors not propagated.
4. Concurrency: race conditions, missing synchronisation, deadlocks,
   non-atomic check-then-act patterns.
5. Performance: obvious O(n^2) or worse where O(n) is possible, N+1
   queries, unnecessary allocations in hot loops.
6. Resources: missing close/dispose, missing try-with-resources / with
   blocks, memory leaks.
7. Maintainability: duplication, magic numbers/strings, misleading
   names — only when clearly defective, not as style preference.

Ignore: pure style (formatting, brace style, whitespace), subjective
preferences with no functional impact, theoretical issues without a
concrete trigger, test coverage and documentation completeness (other
hooks own those).

eight shaky ones. Cap your output at MAX_FINDINGS (see <context>);
when you have more candidates than the limit, prioritise by severity
